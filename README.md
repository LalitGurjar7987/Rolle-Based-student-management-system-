The Academic Management System is a web-based application developed using Java, JSP, Servlets, and MySQL to automate and streamline academic operations within an educational institution. The system is designed with a role-based access approach, supporting three types of users: Admin, Teacher, and Student, each with specific permissions and functionalities.

The Admin module is responsible for managing core data, including adding and viewing students, teachers, and subjects. This ensures centralized control over the system and maintains data consistency. The Teacher module allows teachers to access their assigned subjects, view enrolled students, mark attendance, and enter student marks. This helps in efficient academic tracking and reduces manual effort. The Student module provides a personalized dashboard where students can view their subjects, check subject-wise attendance, and access their marks. Additionally, students can view and download their marksheet, which is generated dynamically using data from multiple related tables.

The application follows the MVC (Model-View-Controller) architecture, ensuring a clear separation of concerns. JSP is used for the view layer to handle the user interface, Servlets act as controllers to manage request and response handling, and DAO (Data Access Object) classes are used in the model layer to interact with the database using JDBC. The system also implements session management for authentication and role-based authorization, ensuring that users can only access functionalities permitted to their roles.

Database operations are optimized using PreparedStatement to prevent SQL Injection and improve performance. Complex data retrieval, such as marksheet generation and attendance tracking, is implemented using SQL JOIN queries to combine multiple tables efficiently.

Overall, this project demonstrates practical knowledge of Java web development, database design, role-based access control, and secure coding practices, making it a scalable and real-world applicable solution.
